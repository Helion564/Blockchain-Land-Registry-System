Smart Contract Development
Full Solidity contract with structs, mappings, access control, events

Create contracts/LandRegistry.sol. Key design decisions:

Contract Architecture

  • Land struct — stores all land data on-chain
  • TransferHistory struct — immutable audit trail for every ownership change
  • Mappings — O(1) lookups by landId and wallet address
  • onlyAdmin modifier — access control for write operations
  • Events — emitted on every state change for frontend listening and indexing
  • Pagination — getAllLandIds(start, count) prevents gas-limit hits on large datasets
The full contract code is in contracts/LandRegistry.sol (included in this project). Below is the structural overview:
solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.19;

contract LandRegistry {

  // ── Structs ───────────────────────────────────────────
  struct Land {
    uint256 landId;
    string  ownerName;
    address ownerWallet;
    string  location;    // GPS or text
    string  landType;    // Residential/Agricultural/etc
    uint256 price;       // in Wei
    uint256 timestamp;
    bool    exists;
  }

  struct TransferHistory {
    address fromWallet; string fromName;
    address toWallet;   string toName;
    uint256 timestamp;  string reason;
  }

  // ── State Variables ───────────────────────────────────
  address public admin;
  uint256 public landCount;
  mapping(uint256 => Land) private lands;
  mapping(uint256 => TransferHistory[]) private transferHistories;
  mapping(address => uint256[]) private ownerLands;

  // ── Modifier ──────────────────────────────────────────
  modifier onlyAdmin() {
    require(msg.sender == admin, "Not admin");
    _;
  }

  // ── Events ────────────────────────────────────────────
  event LandAdded(uint256 indexed landId, ...);
  event OwnershipTransferred(uint256 indexed landId, ...);
  event LandUpdated(uint256 indexed landId, ...);

  // ── Admin Functions ───────────────────────────────────
  function addLand(...) external onlyAdmin returns (uint256) { ... }
  function transferOwnership(...) external onlyAdmin { ... }
  function updateLandPrice(...) external onlyAdmin { ... }

  // ── View Functions (public) ───────────────────────────
  function getLand(uint256 _landId) external view returns (...) { ... }
  function getAllLandIds(uint256 _start, uint256 _count) external view { ... }
  function getTransferHistory(uint256 _landId) external view { ... }
}

Gas Efficiency Notes

  • Use private on state variables (saves gas vs public)
  • Paginated reads via getAllLandIds(start, count) — never fetch unbounded arrays
  • Use calldata for string parameters in external functions (cheaper than memory)
  • Optimizer enabled in hardhat.config with runs: 200
  • Only emit events for state changes, not reads

Blockchain Integration
How ethers.js connects frontend to smart contract

The useBlockchain Hook

All blockchain interactions are centralized in hooks/useBlockchain.js. The pattern:

javascript
// 1. Connect MetaMask → get provider + signer
const ethProvider = new ethers.BrowserProvider(window.ethereum);
const ethSigner   = await ethProvider.getSigner();

// 2. Instantiate contract
const contract = new ethers.Contract(
  CONTRACT_ADDRESS,   // from LandRegistry.json
  CONTRACT_ABI,       // from LandRegistry.json
  ethSigner           // enables write operations
);

// 3. READ — no MetaMask popup (free)
const land = await contract.getLand(1);

// 4. WRITE — triggers MetaMask confirmation
const tx      = await contract.addLand(...args);
const receipt = await tx.wait(); // wait for mining

// 5. Parse price from/to ETH ↔ Wei
const priceWei = ethers.parseEther("10.5");   // ETH → Wei
const priceEth = ethers.formatEther(priceWei); // Wei → ETH

Transaction Flow

  1. User fills form and clicks submit
  2. useBlockchain calls the contract write function
  3. MetaMask popup appears — user confirms & pays gas
  4. tx.wait() waits for Ganache to mine the block (~1s)
  5. Receipt returned → toast notification shown
  6. refreshLands() re-reads all land data

Event Listening (Optional Enhancement)

javascript
// Listen for LandAdded events in real-time
contract.on("LandAdded", (landId, ownerName, ownerWallet) => {
  console.log(`New land #${landId} added for ${ownerName}`);
  refreshLands(); // auto-refresh UI
});

// Cleanup on unmount
return () => contract.removeAllListeners();

Map Component

javascript
import { GoogleMap, Marker, InfoWindow, useLoadScript } from '@react-google-maps/api';

function LandMap({ lands }) {
  const { isLoaded } = useLoadScript({
    googleMapsApiKey: process.env.REACT_APP_GOOGLE_MAPS_KEY,
  });

  const center = { lat: 20.5937, lng: 78.9629 }; // India center

  if (!isLoaded) return <div>Loading map...</div>;

  return (
    <GoogleMap zoom={5} center={center} mapContainerStyle={{width:'100%', height:'500px'}}>
      {lands.map(land => {
        // Parse "28.6139°N, 77.2090°E" to {lat, lng}
        const coords = parseCoords(land.location);
        if (!coords) return null;
        return (
          <Marker key={land.landId} position={coords}
            title={`#${land.landId}: ${land.ownerName}`} />
        );
      })}
    </GoogleMap>
  );
}

function parseCoords(location) {
  // Regex: extract lat/lng from "28.6139°N, 77.2090°E"
  const m = location.match(/(\d+\.\d+)°([NS]),?\s*(\d+\.\d+)°([EW])/);
  if (!m) return null;
  return {
    lat: m[2] === 'N' ? +m[1] : -m[1],
    lng: m[4] === 'E' ? +m[3] : -m[3],
  };
}
Store your Maps API key in frontend/.env as REACT_APP_GOOGLE_MAPS_KEY=your_key_here. Never commit this file.

IPFS Document Storage Optional
Store land deed PDFs on IPFS, save the CID on-chain

Why IPFS?

Storing files directly on Ethereum is expensive. IPFS stores the file, and the blockchain stores the unique content hash (CID) — giving you immutable, decentralized file references cheaply.

Setup with Pinata (Recommended)

bash
cd frontend
npm install @pinata/sdk axios

Upload & Store Pattern

javascript
// 1. Upload file to IPFS via Pinata API
async function uploadToIPFS(file) {
  const formData = new FormData();
  formData.append('file', file);

  const res = await axios.post(
    'https://api.pinata.cloud/pinning/pinFileToIPFS',
    formData,
    { headers: {
        'pinata_api_key':    process.env.REACT_APP_PINATA_KEY,
        'pinata_secret_api_key': process.env.REACT_APP_PINATA_SECRET,
    }}
  );
  return res.data.IpfsHash; // CID e.g. "QmXyz..."
}

// 2. Add ipfsHash field to the Land struct in Solidity
struct Land {
  ...existing fields...
  string ipfsHash; // ← add this
}

// 3. Retrieve and display the document
const ipfsUrl = `https://gateway.pinata.cloud/ipfs/${land.ipfsHash}`;
// Render as <a href={ipfsUrl}>View Deed</a>
